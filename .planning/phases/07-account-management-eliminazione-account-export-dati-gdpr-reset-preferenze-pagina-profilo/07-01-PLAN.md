Create supabase/functions/delete-account/index.ts as a Deno Edge Function.

The function must:

1. Use `import { serve } from "https://deno.land/std@0.177.0/http/server.ts"` and `import { createClient } from "https://esm.sh/@supabase/supabase-js@2.49.4"` — same versions as send-reminders.

2. Accept only POST requests. Reject any other method with 405.

3. Extract the Bearer JWT from the Authorization header. If absent, return 401 with JSON `{ error: "Unauthorized" }`.

4. Create TWO clients:
   - `userClient` = createClient(SUPABASE_URL, SUPABASE_ANON_KEY) — use this to validate the JWT via `userClient.auth.getUser(jwt)`. If getUser returns an error or no user, return 401.
   - `adminClient` = createClient(SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY) — use this for all delete operations.
   - SUPABASE_ANON_KEY is available as Deno.env.get("SUPABASE_ANON_KEY") — it is injected automatically alongside SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY.

5. Extract `userId` from the validated user object.

6. Delete user data in this order (all using adminClient, all using `.eq("user_id", userId)`):
   - `adminClient.from("notifications").delete().eq("user_id", userId)` — skip if error (table may not have user_id column; check: notifications are global, so skip this delete step)
   - `adminClient.from("push_subscriptions").delete().eq("user_id", userId)`
   - `adminClient.from("preferences").delete().eq("user_id", userId)`
   - `adminClient.from("weekly_plan").delete().eq("user_id", userId)`

   Note on notifications: The notifications table is a shared catalog (no user_id column per supabase.ts `fetchNotifications` which does not filter by user_id). Do NOT attempt to delete from notifications.

7. Call `adminClient.auth.admin.deleteUser(userId)`. If this returns an error, return 500 with `{ error: "Failed to delete auth user" }`.

8. Return 200 with `{ success: true }`.

9. Wrap everything in try/catch. On unexpected error, return 500 with `{ error: "Internal server error" }`.

10. Set CORS headers on all responses: `Access-Control-Allow-Origin: *`, `Content-Type: application/json`.
Handle OPTIONS preflight: return 200 with the CORS headers.
