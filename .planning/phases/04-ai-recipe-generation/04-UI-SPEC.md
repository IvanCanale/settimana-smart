### 1. Bell Icon Button (in AppHeader)

**Position:** Immediately left of the existing `ūüĎ§` profile button, in the `justifyContent: "flex-end"` row at `gridColumn: "1 / -1"`.

**Anatomy:**
- Outer `<button>`: 40√ó40px, `borderRadius: "50%"`, `background: var(--cream)`, `border: 1.5px solid rgba(61,43,31,0.12)` ‚ÄĒ exactly matching profile button style
- Icon: `Bell` (lucide, size 20) when no unread; `BellDot` (lucide, size 20) when unread ‚Č• 1
- Unread dot (alternative to BellDot): 8√ó8px circle, `background: var(--terra)`, `position: absolute`, `top: 6px`, `right: 6px` ‚ÄĒ only visible when `unreadCount > 0`
- `title` attribute: `"Notifiche"` (tooltip/aria)

**States:**

**Interaction:** `onClick` calls `onNotificationOpen` prop ‚ÄĒ same prop pattern as `onProfileOpen` in `AppHeaderProps`.

### 3. NuoveRicettePage

**Layout:** Full-width view replacing current tab content, or rendered as an overlay page within the tab shell. Uses the same `max-width` container as other tabs.

**Focal point:** Primary visual anchor is the recipe grid. Eye-entry point is the Playfair Display page title "Ricette della settimana" at the top of the view ‚ÄĒ the display font draws the eye first, then flows down into the grid.

**Anatomy:**
- Page header: back arrow button (left) + page title "Ricette della settimana" in Playfair Display 22px 700 `var(--sepia)` (centered or left-aligned matching tab headings)
- Subtitle: "N ricette aggiunte negli ultimi 7 giorni" ‚ÄĒ 14px 400 `var(--sepia-light)`
- Recipe grid: single column on mobile, 2-column grid `gap: 16px` on ‚Č•640px
- Recipe card: use existing `.recipe-card` class (`border: 1.5px solid rgba(61,43,31,0.12)`, `borderRadius: 16px`, `padding: 12px 16px`, `background: var(--warm-white)`)

**Recipe card anatomy (NuoveRicette variant):**
- Recipe title: 16px weight 700 `var(--sepia)`, line-height 1.4
- Diet tag: `.tag-pill` or `.tag-pill-terra` (existing classes) ‚ÄĒ shows diet type
- Time badge: `.badge-time` (existing class) ‚ÄĒ e.g. "30 min"
- Source attribution: "Fonte: giallozafferano.it" ‚ÄĒ 13px weight 400 `var(--sepia-light)` (from `source_url` domain)
- Wishlist button: 44√ó44px touch target, right-aligned, heart icon ‚ÄĒ idle: `Heart` lucide outline `color: var(--sepia-light)`; active: `HeartFilled` or `Heart` filled `color: var(--terra)`, `background: rgba(196,103,58,0.08)`
- "Aggiunto!" confirmation: inline label 13px `var(--olive)` that appears for 2s then fades ‚ÄĒ replaces heart icon temporarily

**Wishlist interaction:**
- Tap heart ‚Üí optimistic toggle (instant visual feedback, no spinner)
- If toggled ON: heart fills `var(--terra)`, "Aggiunto!" label shown for 2s, recipe stored in `preferences.wishlistedRecipeIds`
- If toggled OFF: heart returns to outline, recipe removed from `preferences.wishlistedRecipeIds`
- No destructive confirmation needed (reversible action)

## Loading & Async States

### Bell icon loading
- While `useNotifications` is loading: show `Bell` (not `BellDot`) ‚ÄĒ no spinner on the header icon
- On error: show `Bell` with no badge ‚ÄĒ fail silently, do not break AppHeader

### NotificationDrawer loading
- While notifications fetch in-flight: show 2 skeleton items ‚ÄĒ `height: 72px`, `background: var(--cream-dark)`, `borderRadius: 12px`, animated `pulse` keyframe (already defined in `globals.css`)
- Fetch timeout (>5s): show error state copy (see Copywriting above)

### NuoveRicettePage loading
- Show 4 skeleton recipe cards during fetch: match `.recipe-card` dimensions, `background: var(--cream-dark)`, `animation: pulse 1.4s ease infinite`
- Loading label below skeleton grid: "Caricamento ricette..." ‚ÄĒ 13px 400 `var(--sepia-light)` centered

### Recipe fetch from Supabase (plan generation)
- Success: plan generates normally ‚ÄĒ no UI change visible
- Loading (recipes not yet in cache): show existing plan skeleton (already handled by `usePlanEngine` loading state)
- Error (Supabase unreachable + no cache): show `.alert-banner` (existing class) inside `PlannerTab` with copy: "Impossibile caricare il catalogo. Il piano user√† le ricette salvate localmente."
