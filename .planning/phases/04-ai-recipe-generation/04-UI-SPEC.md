## Component Contracts

### 1. Bell Icon Button (in AppHeader)

**Position:** Immediately left of the existing `👤` profile button, in the `justifyContent: "flex-end"` row at `gridColumn: "1 / -1"`.

**Anatomy:**
- Outer `<button>`: 40×40px, `borderRadius: "50%"`, `background: var(--cream)`, `border: 1.5px solid rgba(61,43,31,0.12)` — exactly matching profile button style
- Icon: `Bell` (lucide, size 20) when no unread; `BellDot` (lucide, size 20) when unread ≥ 1
- Unread dot (alternative to BellDot): 8×8px circle, `background: var(--terra)`, `position: absolute`, `top: 6px`, `right: 6px` — only visible when `unreadCount > 0`
- `title` attribute: `"Notifiche"` (tooltip/aria)

**States:**
- Default (no unread): `Bell` icon, `color: var(--sepia-light)`
- Unread present: `BellDot` icon, `color: var(--terra)`
- Hover: `background: var(--cream-dark)`, transition 0.15s — matching profile button hover
- Active/open: `background: rgba(196,103,58,0.08)`, `border-color: var(--terra-light)`

**Interaction:** `onClick` calls `onNotificationOpen` prop — same prop pattern as `onProfileOpen` in `AppHeaderProps`.

---

### 2. NotificationDrawer

**Pattern:** Identical slide-in structure to `ProfileDrawer.tsx`. Re-use backdrop + panel layout verbatim.

**Anatomy:**
- Backdrop: `position: fixed`, `inset: 0`, `zIndex: 200`, `background: rgba(61,43,31,0.6)`, `backdropFilter: blur(6px)` — identical to ProfileDrawer
- Panel: `position: fixed`, `top: 0`, `right: 0`, `bottom: 0`, `width: min(400px, 90vw)`, `background: var(--warm-white)`, `overflowY: auto`, `padding: 24px`, `boxShadow: -8px 0 30px rgba(61,43,31,0.15)`, `zIndex: 201`
- Panel header: Playfair Display 22px 700 title "Notifiche" + `×` close button (same style as ProfileDrawer header)
- Notification list: vertically stacked items, `gap: 8px`

**Notification item anatomy (type: `new_recipes`):**
- Container: `background: var(--cream)`, `borderRadius: 12px`, `padding: 14px 16px`, `cursor: pointer`, `transition: all 0.15s`
- Icon area: 38×38px `.section-icon` (existing class) with `Bell` lucide icon, `color: var(--terra)`
- Title: "N nuove ricette disponibili" — 16px weight 700 `var(--sepia)`
- Body: "Questa settimana sono state aggiunte N ricette al catalogo. Scoprile!" — 14px weight 400 `var(--sepia-light)`, line-height 1.5
- Timestamp: "Settimana W12 · 2026" — 12px weight 600 `var(--sepia-light)`, uppercase, `letterSpacing: 0.05em`
- Unread indicator: left border `3px solid var(--terra)` on the container when `read: false`; no border when `read: true`

**States:**
- Unread item: `border-left: 3px solid var(--terra)`, background `var(--cream)`
- Read item: no left border, background `rgba(61,43,31,0.03)` (slightly dimmed)
- Hover: `border-color: var(--terra-light)`, `transform: translateY(-1px)` — matching `.recipe-card:hover`
- Empty state: centered illustration area (see Copywriting section)

**Interaction:**
- Click item → close drawer → navigate to NuoveRicettePage (or open as a new tab view)
- Close button (`×`) → `onClose()`
- Backdrop click → `onClose()`
- Items auto-marked read when drawer opens (optimistic update)

### 3. NuoveRicettePage

**Layout:** Full-width view replacing current tab content, or rendered as an overlay page within the tab shell. Uses the same `max-width` container as other tabs.

**Anatomy:**
- Page header: back arrow button (left) + page title "Ricette della settimana" in Playfair Display 22px 700 `var(--sepia)` (centered or left-aligned matching tab headings)
- Subtitle: "N ricette aggiunte negli ultimi 7 giorni" — 14px 400 `var(--sepia-light)`
- Recipe grid: single column on mobile, 2-column grid `gap: 16px` on ≥640px
- Recipe card: use existing `.recipe-card` class (`border: 1.5px solid rgba(61,43,31,0.12)`, `borderRadius: 16px`, `padding: 14px 16px`, `background: var(--warm-white)`)

**Recipe card anatomy (NuoveRicette variant):**
- Recipe title: 16px weight 700 `var(--sepia)`, line-height 1.4
- Diet tag: `.tag-pill` or `.tag-pill-terra` (existing classes) — shows diet type
- Time badge: `.badge-time` (existing class) — e.g. "30 min"
- Source attribution: "Fonte: giallozafferano.it" — 11px weight 500 `var(--sepia-light)` (from `source_url` domain)
- Wishlist button: 44×44px touch target, right-aligned, heart icon — idle: `Heart` lucide outline `color: var(--sepia-light)`; active: `HeartFilled` or `Heart` filled `color: var(--terra)`, `background: rgba(196,103,58,0.08)`
- "Aggiunto!" confirmation: inline label 12px `var(--olive)` that appears for 2s then fades — replaces heart icon temporarily

**Wishlist interaction:**
- Tap heart → optimistic toggle (instant visual feedback, no spinner)
- If toggled ON: heart fills `var(--terra)`, "Aggiunto!" label shown for 2s, recipe stored in `preferences.wishlistedRecipeIds`
- If toggled OFF: heart returns to outline, recipe removed from `preferences.wishlistedRecipeIds`
- No destructive confirmation needed (reversible action)

## Loading & Async States

### Bell icon loading
- While `useNotifications` is loading: show `Bell` (not `BellDot`) — no spinner on the header icon
- On error: show `Bell` with no badge — fail silently, do not break AppHeader

### NotificationDrawer loading
- While notifications fetch in-flight: show 2 skeleton items — `height: 72px`, `background: var(--cream-dark)`, `borderRadius: 12px`, animated `pulse` keyframe (already defined in `globals.css`)
- Fetch timeout (>5s): show error state copy (see Copywriting above)

### NuoveRicettePage loading
- Show 4 skeleton recipe cards during fetch: match `.recipe-card` dimensions, `background: var(--cream-dark)`, `animation: pulse 1.4s ease infinite`
- Loading label below skeleton grid: "Caricamento ricette..." — 13px 400 `var(--sepia-light)` centered

### Recipe fetch from Supabase (plan generation)
- Success: plan generates normally — no UI change visible
- Loading (recipes not yet in cache): show existing plan skeleton (already handled by `usePlanEngine` loading state)
- Error (Supabase unreachable + no cache): show `.alert-banner` (existing class) inside `PlannerTab` with copy: "Impossibile caricare il catalogo. Il piano userà le ricette salvate localmente."
